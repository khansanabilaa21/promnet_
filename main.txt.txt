GIT merupakan singkatan dari Group Inclusive Tour, yaitu version control system yang digunakan programmer bertujuan berkolaborasi dalam membuat atau mengembangkan aplikasi. GIT bertujuan untuk memudahkan pengelolaan source code program, seperti adanya perubahan pada baris kode yang diganti atau ditambahkan.
Version Control yaitu aplikasi yang berfungsi mengelola dan mencatat setiap perubahan pada file. Setiap perubahan yang terjadi akan dicatat lengkap dengan informasi siapa yang merubah, kapan perubahan dilakukan, dan isi perubahannya. GIT adalah perangkat lunak yang digunakan oleh Linus Torvald untuk membuat kernel Linux pada tahun 2005. Kemudian, GIT dikelola oleh Jumino Hamano, seorang programmer asal Jepang. Git merupakan tool gratis dan dapat diinstal di perangkat dengan sistem operasi Mac, Windows, Linux, hingga Solari.
Fungsi GIT adalah aplikasi yang memuat beragam fungsi yang dapat ditemukan pada saat melakukan coding. Berikut fungsi GIT sebagai berikut: 
1. Dapat bekerja sama dengan programmer lain dalam membuat atau mengembangkan sistem yang setiap programmer akan bekerja di dalam branch-nya masing-masing. Perubahan yang terjadi di dalam code program juga akan dapat mudah dilacak. 
2. Open Source GIT, software yang bersifat open source. Dapat mengakses GIT secara gratis, tanpa biaya.
3. Dapat mengorganisir dengan menggunakan GIT, mampu membuat folder project berdasarkan versinya, seperti project v1, v2, dan seterusnya. Kemudian, dapat memiliki 1 project dengan menggunakan database khusus yang berisi semua file versi projek.
4. Platform yang memiliki fleksibilitas, terdapat banyak aplikasi GIT yang dapat digunakan, seperti Gitlab, GitHub, Bitbucked, SourceForge. Git merupakan  platform yang memiliki fleksibilitas, dapat digunakan untuk mengerjakan berbagai projek bersama tim.
5. Memiliki fungsi untuk backup, yang berfungsi dapat dengan mudah melakukan restore script program kembali pada keadaan semula. Berikut istilah-istilah dasar yang terdapat pada GIT:
a. Repository: database yang menyimpan riwayat perubahan. 
b. Snapshot : potret kondisi file atau folder. 
c. Commit : snapshot yang disimpan pada repository. 
d. Pull : proses menarik perubahan dari repository remote ke repository lokal. 
e. Push : mendorong perubahan dari repository lokal ke repository remote. 
f. Master : nama Branch default yang terbuat saat membuat Repository. 
g. Branch : rangkaian Commit yang saling berkaitan. Ditemukan pada setiap programmer yang tergabung dalam mengerjakan suatu projek, bekerja pada branch-nya masing-masing. 
h. Head : ujung Branch yang merupakan commit terbaru. 
i. HEAD : head yang sedang aktif. Meskipun pada satu Repository dapat memiliki banyak Branch, namun hanya satu yang aktif. 
j. Working Folder : folder yang merupakan tempat kita bekerja. GIT dapat memantau dan mengetahui file dan folder mana yang dilakukan perubahan. 
k. Stagging : fitur unik di GIT yang tidak ada di version control lain. Dengan staging area, bertujuan dapat memilih perubahan mana yang akan di commit dan yang tidak. 
l. Object Store : database tempat semua Commit disimpan. 

Berikut keunggulan dari Git, yaitu:
- Design yang sederhana
- Mendukung dalam pengembangan non-linear (pengembangan paralel)
- Sistem terdistribusi, peer to peer
- Mendukung dalam proyek besar seperti Kernel Linux
- Akses menggunakan command line (tidak harus client-server)
- Penyimpanan murni berbasis file (tidak menggunakan database/SQL)

Berikut kelemahan dari Git, yaitu:
- Tidak berjalan optimal untuk pengembang tunggal
- Dukungan untuk Windows terbatas dibandingkan Linux
